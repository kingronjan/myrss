## 文档编写规范

### 文件命名
- 使用英文命名，全部大写扩展名
- 文件名应清晰反映内容
- 保持一致的命名风格

### 内容结构
- 使用 Markdown 标题层级
- 添加目录索引（如果文档较长）
- 包含版本和更新日期

### 代码示例
- 使用正确的语法高亮
- 提供必要的上下文说明
- 保持示例的准确性和可运行性

### 链接引用
- 文档间相互引用使用相对路径
- 外部链接明确标注
- 避免死链接

## 贡献指南

### 文档贡献流程
1. **发现问题**: 识别文档中的错误、遗漏或不清晰之处
2. **创建修改**: 在本地修改相应的文档文件
3. **提交更改**: 提交 Pull Request 并描述修改内容
4. **代码审查**: 等待项目维护者审查
5. **合并发布**: 通过审查后合并到主分支

### 文档质量标准
1. **准确性**: 内容必须准确反映项目实际情况
2. **完整性**: 覆盖所有重要方面，无重大遗漏
3. **清晰性**: 表达清晰，易于理解
4. **一致性**: 与其他文档保持一致
5. **实用性**: 对读者有实际帮助
